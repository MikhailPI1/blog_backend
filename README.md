## 🏗 Архитектура

![Архитектура проекта](docs/architecture.png)

### Ключевые компоненты:
- **API Layer** - FastAPI приложение с роутами и схемами Pydantic
- **Cache Layer** - Redis для кеширования часто запрашиваемых данных
- **Database Layer** - PostgreSQL с хранимыми процедурами
- **Business Logic** - SQL функции для операций с данными

## 🛠 Технологии

- **FastAPI**
- **PostgreSQL 15**
- **Redis 7**
- **Docker & Docker Compose**
- **Pytest**

## 📁 Структура проекта
```bash
├── api/ # FastAPI приложение
│ ├── src/
│ │ ├── include/
│ │ │ ├── cach/ # Python обертка над Redis
│ │ │ ├── db/ # Python обертка над Postgre
│ │ │ ├── log/ # Логгер
│ │ │ ├── routers/
│ │ │ └── schemas/
│ │ └── main.py
│ ├── tests/ # Тесты
│ └── requirements.txt
├── db/ # SQL скрипты
│ ├── 1table.sql # Создание таблиц
│ ├── 2index.sql # Индексы
│ ├── 3function.sql # Хранимые функции
│ └── 4data.sql # Тестовые данные
├── docker-compose.yml # Оркестрация контейнеров
├── .env.example # Пример переменных окружения
└── README.md
```

Основная нагрузка ложится на БД слой в которой заключены все функции работающие с данными (sql выполняет операции внутри себя быстрее, чем он бы делал через ORM преобразования). Следующим идет слой оберток над Redis и BD, которые взаимозаменяют друг друга, принцип декорирования через redis обертки позволяет снизить нагрузку на бд за счёт быстрого поиска данных в кеше. Это легко логируется + добавляет прозрачности. В конце все обернуто в presentation слой на FastAPI.

При работе кешем из Redis разработаны декораторы для валидирования и инвалидирования данных соответсвенно.

Тестовые данные для запросов находятся в db/4data (автоматически установятся при развертывании).

## Запуск проект

```bash
#Для запуска проекта необходимо ввести следующую команду:
docker-compose up -d

#Для остановки
docker-compose down -v # Для очистки данных
```

Docker-compose собран так, чтобы сервер разворачивался после развертывания Redis и Postgre. ДАнные кешируются на уровне Dockera, поэтому скорость повторного запуска контейнера сильно снижена.


## Погрешности

На данный момент полностью функционируют следующие роуты:
-user
-post
-tags
-categories

Роуты требующие доработки:
-post_tags
-post_views

Для них написаны функции и таблцы в sql, но на уровне Service layer и Presentation layer требуется доработки и исправления ошибок.
